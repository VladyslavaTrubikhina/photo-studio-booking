# Reading guide

**Application guide** - helper to run the application

**Functional design** - describes application functionality. Which roles the application has and what permissions these roles have

**Requirements** - describes what user stories are for each user role and what is required in the application to what extent according these user stories

**Technical design** - describes how the application is designed. What technologies are used and how the database is designed

**Users credentials** - contains login credentials for initial users of the application (1 admin and 2 normal users)
